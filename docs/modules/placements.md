# Placements

## What This Module Is For
Placements tracks accepted and non-accepted hiring outcomes tied to submission flow.

## Placement Types
- Temporary
- Permanent

## Compensation Types
- Hourly
- Daily Rate
- Salary

Field requirements adjust based on selected type.

Examples:
- Hourly: RT/OT bill+pay fields
- Daily Rate: bill+pay daily fields
- Salary: annual compensation fields

## Currency
Currency selection supports:
- USD
- CAD

Currency values are formatted during entry and in list displays.

## Date Fields
- Offered On (required)
- Start Date (required)
- End Date (optional)

## Lock Rules
- Accepted placements become read-only.
- Related submission can also be locked accordingly.

## Actions Menu
Includes withdraw/cancel controls with confirmation and reason capture.

## Best Practice
1. Match compensation model to actual contract type.
2. Record reason when withdrawing or cancelling.
3. Keep start/end dates accurate for reporting and payroll handoff.
